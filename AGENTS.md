# Lampa-Plex — контекст проекта

## Проект

Репозиторий: `KZuev/Lampa-Plex` (публичный).
Основной файл: `lampa-plex.js` — плагин Plex для медиа-интерфейса Lampa (3.0+, `Lampa.Maker`/`Lampa.Component`/`Lampa.SettingsApi`).
Общение с пользователем — **на русском**.

Плагин добавляет пункт бокового меню «Медиатека Plex» (список библиотек Plex → сетка карточек фильмов/сериалов), раздел настроек «Plex» (подключение к серверу, выбор библиотек, синхронизация просмотра) и запускает воспроизведение файлов с личного Plex-сервера через `Lampa.Player` (Direct Play).

Разрабатывался по аналогии с соседним плагином [`KZuev/LampaTrakt`](https://github.com/KZuev/LampaTrakt) — оттуда взяты общие паттерны (single-file плагин, `Lampa.SettingsApi`, device-flow авторизация с QR-кодом, hybrid-интеграция с родной карточкой Lampa через `Lampa.Listener.follow('full', ...)`, синхронизация прогресса через `Lampa.Timeline`).

Коммитим и пушим **напрямую в `main`**, без PR (так же, как в LampaTrakt) — если явно не попросят иначе.

## Публикация

`.github/workflows/pages.yml` при каждом пуше в `main` публикует корень репозитория на GitHub Pages (`actions/upload-pages-artifact` + `actions/deploy-pages`), без `enablement`/повышенных прав у токена — Pages должен быть один раз включён вручную владельцем репозитория (Settings → Pages → Source: **GitHub Actions**). Ссылка для установки в Lampa: `https://kzuev.github.io/Lampa-Plex/lampa-plex.js`.

**Важно:** не добавлять в workflow разрешение `administration: write` или `enablement: true` ради автоматического включения Pages — это расширение прав CI-токена до уровня администратора репозитория уже один раз блокировалось классификатором auto-mode как самовольное повышение привилегий. Включение Pages — осознанное действие владельца репозитория, а не то, что должен делать CI.

## Архитектура плагина (`lampa-plex.js`)

Один файл, IIFE, ES5-совместимый синтаксис, без сборки. Разделы внутри файла (сверху вниз):

- **Утилиты** — `extend`, `escapeHtml`, `buildQueryString`.
- **Хранилище** (`Lampa.Storage`): `plex_server_url`, `plex_token`, `plex_client_id` (генерируется автоматически, используется как `X-Plex-Client-Identifier`), `plex_sections_selected` (JSON-массив ключей выбранных библиотек), `plex_sync_enabled` (trigger), `plex_tmdb_index` + `plex_tmdb_index_updated_at` (персистентный индекс `tmdbId → ratingKey` всей медиатеки, см. ниже).
- **HTTP-слой**: `plexRequest`/`plexUrl` — запросы к Plex Media Server, все параметры авторизации (`X-Plex-Token` и т.д.) передаются **в query-строке**, не заголовками (чтобы не упираться в CORS-preflight на произвольных локальных адресах серверов). `plexTvGet`/`plexTvUrl` — запросы к `plex.tv` (PIN-логин, список серверов).
- **`Api`** — объект с методами `identity`, `sections`, `list` (постранично + поиск через `/library/sections/{key}/search`), `metadata`, `children` (сезоны/серии).
- **Вход через Plex.tv**: `startPlexTvLogin` → `POST /api/v2/pins` → модалка с QR (как device-auth в LampaTrakt) → поллинг `GET /api/v2/pins/{id}` → `discoverServers` (`GET /api/v2/resources`) → выбор сервера при нескольких → `applyServer` сохраняет токен+URL.
- **Настройки** (`Lampa.SettingsApi`, component `'plex'`): адрес сервера/токен вручную (`Lampa.Input.edit`, паттерн 1:1 как `trakt_client_id`/`trakt_client_secret` в LampaTrakt), кнопка входа через Plex.tv, проверка соединения, выбор библиотек (`Lampa.Select.show` с чекбоксами через повторный рендер; если `getSections()` ещё пуст — при открытии пикера `selected` предзаполняется всеми ключами из `Api.sections()`, чтобы UI визуально отражал реальное поведение по умолчанию «показываются все библиотеки», а не выглядел как «ничего не выбрано»), тумблер синхронизации, сброс настроек.
- **Меню**: `Lampa.Menu.addButton` (публичный API, а не ручной jQuery-хак в `.menu__list`, который использует LampaTrakt для совместимости со старыми версиями Lampa). Если Plex не настроен — клик по пункту меню открывает настройки плагина (`Lampa.Controller.toggle('settings'); Lampa.Settings.create('plex')`).
- **Компонент `plex_library`** (`Lampa.Component.add`) — единственный кастомный Activity-компонент, тонкая обёртка над штатным `Lampa.Maker.make('Category', object)` (тот же класс, что рендерит нативные каталоги TMDB) с собственными `onCreate`/`onNext`/`onInstance`. Карточки нормализуются в `toCard()` в минимальный набор полей, подтверждённый рабочим по коду LampaTrakt (`component`, `source`, `method`, `id`, `title`, `poster`, `image`, `release_date`, `vote_average`).
- **Hybrid-роутинг карточки**: если у элемента Plex есть `Guid` с `tmdb://` (или legacy `guid` вида `themoviedb://...`) — карточка помечается `component:'full', source:'tmdb'` и `onEnter` делает `Lampa.Activity.push(element)`, открывая родную карточку Lampa. Иначе `onEnter` вызывает `openPlexDetailModal` — свою модалку (постер/описание/год из Plex + кнопка Play; для сериала — сезон/серия через `Lampa.Select.show`).
- **Индекс медиатеки** (`_plexTmdbIndex`, `rebuildTmdbIndex()`): персистентная карта `method:tmdbId → {ratingKey}`, обходит постранично (`fetchAllSectionItems`, размер страницы 200) все выбранные секции через `Api.list()`. Загружается из `Storage` при старте плагина, обновляется: (1) тихо при **каждом** запуске Lampa (`autoRebuildTmdbIndexOnStart`, отложен на 3с через `setTimeout`, чтобы не мешать старту приложения — без проверки на устаревание, безусловно), (2) сразу после изменения списка библиотек в `pickSections()` (только если набор реально отличается от того, что было при открытии экрана — см. `arraysEqualUnordered`), (3) вручную кнопкой «Обновить кэш медиатеки» в настройках (со статусом «когда обновлялось / сколько сопоставлено»). `toCard()` при построении сетки `plex_library` также дополняет этот же объект в памяти — обзор через «Медиатека Plex» мгновенно виден и до плановой пересборки.
- **Кнопка «Смотреть в Plex» на родной карточке**: `Lampa.Listener.follow('full', ...)` на событии `'complite'` смотрит в `_plexTmdbIndex[method+':'+id]` (не зависит от того, как пользователь попал на карточку — поиск, рекомендации, наша сетка — лишь бы индекс был собран хоть раз) и, если совпадение есть, вставляет кнопку в `.buttons--container` — тот самый контейнер, откуда меню «Смотреть» строит список источников (Торренты/Онлайн/…), обычным `.append()`. Класс кнопки `full-start__button selector` (+ маркерный `plex-watch-btn`), внутри — инлайновый `PLEX_PLAY_ICON` (отдельная константа, не `PLEX_ICON` из меню/настроек — см. запись в истории версий про новый значок) и `<span>` с подписью, обработчик — `.on('hover:enter', ...)`. Паттерн проверен и скопирован напрямую из `LampaTrakt` — функция `addAtButton(card, method)` создаёт кнопку «Авто-торрент» с классом `full-start__button selector trakt-magic-button`, а `onFullCardReady` вставляет её через `magicRoot.find('.buttons--container').append(atBtn)` (комментарий в их коде: «вставляем в `.buttons--container`, откуда «Смотреть» берёт список источников»). Более старая версия нашей кнопки вставлялась отдельным чипом после `.full-start-new__rate-line` и зависела от per-session `_plexTmdbMatchCache`, заполнявшегося только при просмотре собственной сетки плагина — оба ограничения устранены (см. запись в истории версий ниже).
- **Автозакрепление через штатный механизм Lampa** (`pinPlexAsDefaultSource`/`restorePlexPriorityIfNeeded`, оба в `initFullCardHook`): изучен `src/components/full/start/buttons.js` из `yumata/lampa-source` — родной «пин» источника хранится ГЛОБАЛЬНО в `Lampa.Storage` под ключом `full_btn_priority`, значение — `Lampa.Utils.hash(...)` от `outerHTML` конкретной кнопки (без класса `focus`); `onGroupButtons()` при каждом `emit('groupButtons')` ищет среди `.buttons--container > .full-start__button` кнопку с совпадающим хэшем и, если находит, клонирует её через `emit('priorityButton', btn)` в `.full-start-new__buttons` как `.button--priority` (это и есть визуально «закреплённая на карточке» кнопка). Критичная деталь тайминга — в `components/full.js`: `Lampa.Listener.send('full', {type:'complite', ...})` вызывается, и **на следующей же строке**, синхронно, `this.items[0].emit('groupButtons')` — то есть если внутри нашего обработчика `'complite'` мы синхронно вставили кнопку в DOM и выставили `full_btn_priority` в её хэш, `onGroupButtons()` гарантированно найдёт и закрепит именно её, без необходимости лезть во внутренний `Emit` инстанса `Start` (недоступен снаружи). `plexButtonHash()` — хэш строит один раз от статичной разметки кнопки и кэширует (разметка одинакова для любой карточки → хэш одинаков). Сохранение/восстановление предыдущего состояния — отдельный ключ `plex_prev_btn_priority`: `pinPlexAsDefaultSource()` сохраняет туда текущее значение `full_btn_priority` ТОЛЬКО если оно ещё не равно нашему хэшу (не затирает исходное предпочтение при переходе между несколькими Plex-карточками подряд), затем выставляет наш хэш; `restorePlexPriorityIfNeeded()` (вызывается в ветке «нет совпадения» в `initFullCardHook`, а также в `resetSettings()`) возвращает сохранённое значение обратно, но только если `full_btn_priority` всё ещё равен нашему хэшу (если пользователь сам вручную закрепил что-то другое уже после нас — не перезатираем его недавний осознанный выбор).
- **Воспроизведение** — только Direct Play: `Api.metadata(ratingKey)` → `Media[0].Part[0].key` → прямая ссылка с токеном в query. Субтитры — только sidecar-дорожки (`Stream.streamType === 3`), embedded/PGS не извлекаются. `Lampa.Timeline.update`+`Lampa.Timeline.view(hash)` (`hash = 'plex_' + ratingKey`) перед `Lampa.Player.play(...)` — резюме позиции. Передача во внешний плеер — целиком на стороне самой Lampa (`Storage.field('player'...)`), плагин не реализует ничего специально для этого.
- **Синхронизация прогресса**: `Lampa.Player.listener.follow('destroy', ...)` читает `Lampa.Timeline.view(hash)` и шлёт `GET /:/timeline` (+ `/:/scrobble` при ≥90%) на сервер Plex.
- **Интеграция с LampaTrakt** (опционально, настройка `plex_trakt_status_enabled`, видна только если LampaTrakt установлен):
  - Детект присутствия: `traktAvailable()` = `Lampa.SettingsApi.getComponent('trakt')` — LampaTrakt регистрирует свой settings-компонент под именем `'trakt'`. Специально НЕ используется `Lampa.Manifest.plugins` (как это делает сам LampaTrakt для детекта плагина «Applecation») — проверено, что сам LampaTrakt нигде не пишет `Lampa.Manifest.plugins = {...}` для саморегистрации, поэтому этот путь для него не сработал бы.
  - Учётные данные читаются напрямую из общего `Lampa.Storage` (тот же движок хранения, что и у LampaTrakt, всё в одном Lampa-приложении): `trakt_client_id` + разрешение активного токена в `getTraktActiveToken()`, повторяющее порядок `multiAccountGetActiveSlot`/`multiAccountGetSlot` из LampaTrakt (`trakt_accounts[].slot === trakt_active_slot`, откат на плоский `trakt_token` для немигрированных установок).
  - `traktRequest()` — прямые вызовы `api.trakt.tv` (заголовки `trakt-api-key`/`Authorization: Bearer`) в обход самого LampaTrakt, т.к. он не предоставляет публичного JS API для чтения своего кэша просмотренного.
  - `getTraktWatchedIndex()` — кэш в памяти (10 минут, `TRAKT_WATCHED_CACHE_MS`) карты `movie:tmdbId`/`tv:tmdbId` из `/sync/watched/movies` + `/sync/watched/shows`. Для сериалов «completed» считается по `plays >= aired_episodes` (та же устойчивая эвристика, что LampaTrakt применяет после того, как Trakt периодически убирает поэпизодную разбивку `seasons` из ответа — см. их же changelog v3.2.36–38), а не по наличию точной разбивки.
  - Отображение — только в собственном интерфейсе плагина (карточка `openPlexDetailModal` через `statusLineForMeta`, список серий `openEpisodePicker`), не на родной карточке Lampa — там статус уже показывает сам LampaTrakt, если установлен и авторизован.
  - `syncTraktStatusesToPlex()` — ручная (кнопка с подтверждением через `Lampa.Select.show`) односторонняя запись Trakt → Plex: фильмы и **полностью** просмотренные сериалы (все серии) из истории Trakt отмечаются просмотренными в Plex через `GET /:/scrobble`. Сопоставление — через уже собранный `_plexTmdbIndex`. Последовательно, с задержкой 150мс между запросами (не заваливать Plex-сервер). Частично просмотренные сериалы **намеренно пропускаются** — риск ошибки из-за нестабильной поэпизодной разбивки Trakt сочтён неприемлемым для необратимой записи в Plex.

## Известные ограничения (см. также README)

- Только Direct Play — нет серверного транскодирования Plex (HLS-сессии, `/video/:/transcode/universal/*`). Файлы в несовместимых кодеках (HEVC/DTS и т.п.) не воспроизводятся, если ни встроенный, ни внешний плеер их не тянут напрямую.
- Субтитры — только внешние (sidecar), не встроенные в контейнер.
- Синхронизация прогресса при внешнем плеере зависит от того, возвращает ли конкретный внешний плеер позицию обратно в `Lampa.Timeline` — как и для любых других источников в Lampa, это вне контроля плагина.
- Одна выбранная секция на попытку сохранения нескольких Plex-серверов на один аккаунт не тестировалась специально; при входе через plex.tv можно выбрать сервер из списка, переключение — через повторный вход или ручную правку адреса.

## История версий

**v1.0.0** — Первая версия. Настройки Plex (сервер/токен вручную + вход через plex.tv), пункт меню «Медиатека Plex», компонент `plex_library` на `Lampa.Maker.make('Category', ...)`, hybrid-интеграция с родной карточкой (кнопка «Смотреть в Plex»), собственная модалка для несопоставленного с TMDB контента + сезон/серии через `Select`, Direct Play через `Lampa.Player.play`, синхронизация прогресса с Plex (`/:/timeline`, `/:/scrobble`).

**Публикация на GitHub Pages** — добавлен `.github/workflows/pages.yml` (копия воркфлоу LampaTrakt). Первый прогон упал (`Get Pages site failed... Not Found`) — Pages не был включён в настройках репозитория; включили вручную (Settings → Pages → Source: GitHub Actions), повторный запуск (пустой коммит) прошёл успешно. Попытка исправить это добавлением `enablement: true` + `administration: write` в workflow была заблокирована auto-mode классификатором как самовольное расширение прав CI-токена — оставлено как есть, без auto-enablement.

**Иконка плагина** — значок в левом меню и в настройках заменён на настоящий логотип Plex (инлайн SVG, один `path fill="currentColor"`, путь из проекта Simple Icons, CC0) — по аналогии с `TRAKT_ICON` у LampaTrakt.

**Кнопка «Смотреть в Plex» → обычный источник + персистентный индекс медиатеки** — по просьбе пользователя приведена в соответствие с тем, как в LampaTrakt сделана кнопка «Авто-торрент»: перенесена из отдельного чипа после `.full-start-new__rate-line` в общий `.buttons--container` (класс `full-start__button selector`, значок Plex вместо текстового «▶»). Одновременно устранена зависимость от истории навигации: раньше кнопка появлялась только на тайтлах, которые пользователь уже пролистал через «Медиатека Plex» в текущей сессии (`_plexTmdbMatchCache` заполнялся исключительно в `toCard()`). Теперь `_plexTmdbIndex` — персистентная в `Lampa.Storage` карта `tmdbId → ratingKey` по всей медиатеке (все выбранные секции, постранично), собирается автоматически при старте (если старше 12 ч), после изменения списка библиотек и вручную кнопкой «Обновить кэш медиатеки» — кнопка теперь показывается на любой карточке независимо от способа перехода на неё, как только индекс собран хотя бы раз.

**Фикс: индекс не собирался, если библиотеки не выбраны явно** — пользователь сообщил, что кнопка по-прежнему не появлялась без захода в «Медиатеку Plex», несмотря на предыдущий фикс. Причина: `rebuildTmdbIndex()`/`maybeAutoRebuildTmdbIndex()` брали секции только из `getSections()` (явный выбор в «Выбрать библиотеки») и молча выходили, если список пуст — а он пуст по умолчанию, пока пользователь ни разу не открывал этот пункт настроек. При этом `openLibrary()` (сама «Медиатека Plex») всегда работала — она при пустом выборе показывает **все** секции сервера, поэтому создавалось впечатление, что «без явного выбора библиотек всё работает, кроме индекса». Добавлена `resolveActiveSectionKeys()` — общая логика (явный выбор пользователя, иначе все секции с сервера через `Api.sections()`), теперь используется и в сборке индекса, а не только в самой «Медиатеке».

**UX-фиксы по итогам обратной связи**:
- Автообновление кэша — раз в 12 часов при старте Lampa (если старше), плюс сразу после сохранения выбора библиотек и вручную кнопкой; не «только по кнопке».
- «Выбрать библиотеки»: чекбоксы теперь по умолчанию показывают **все** библиотеки отмеченными, если пользователь ещё ни разу не сохранял выбор (`selected` в `pickSections()` предзаполняется всеми `Api.sections()`, если `getSections()` пуст) — раньше UI показывал «ничего не выбрано», хотя фактически показывались все библиотеки, что вводило в заблуждение.
- «Смотреть в Plex» теперь вставляется через `.prepend()` и сразу получает фокус (`Controller.collectionFocus`) — «закреплённый»/приоритетный источник в списке, когда контент есть в Plex; если совпадения нет, кнопка не создаётся и родной набор источников/фокус Lampa не меняются.

**Фикс: лишние пересборки/уведомления при простом открытии «Выбрать библиотеки»** — пользователь спросил, можно ли сделать автообновление кэша тихим в фоне при запуске Lampa. Автоматический путь (`maybeAutoRebuildTmdbIndex`, таймер) уже был тихим (`notify: false`) — но `pickSections()` безусловно вызывал `rebuildTmdbIndex({notify:true})` в `onSelect('Готово')`/`onBack`, даже если пользователь просто открыл и закрыл экран без единого изменения — из-за этого казалось, что уведомления сыплются «на пустом месте». Добавлена `arraysEqualUnordered()` и снимок `initialSelected` при открытии пикера: пересборка индекса (и её уведомление) теперь запускается только если набор выбранных библиотек реально отличается от того, что было при открытии экрана.

**Автообновление на каждом запуске + отмена «закреплённой первой» кнопки** — два уточнения от пользователя:

**Фикс: включил «Статусы Trakt.TV» — статусы Plex пропали, статусы Trakt не появились** — пользователь сообщил ровно это. Причина найдена в `getTraktWatchedIndex()`: оба запроса к Trakt (`/sync/watched/movies`, `/sync/watched/shows`) внутри `Promise.all` были обёрнуты каждый в свой `.catch(() => [])`. При реальном сбое (протухший токен LampaTrakt, неверный `client_id`, сетевая ошибка и т.п.) это означало, что `Promise.all` НИКОГДА не отклонялся — он благополучно резолвился пустыми массивами, строился пустой (но полноценный, truthy) индекс, и этот пустой индекс кэшировался на 10 минут как «настоящие данные Trakt». В результате внешний `.catch()` в `statusLineForMeta`/`openEpisodePicker` (задуманный как откат на Plex при сбое) никогда не срабатывал — Trakt-ветка молча использовалась с пустыми данными везде, поэтому статусы Trakt не показывались (индекс пуст), а откат на Plex не происходил (ошибки формально не было). Убраны внутренние `.catch()` у обоих `traktRequest(...)` — теперь реальный сбой корректно долетает до внешнего `.catch()` и включает откат на Plex, как и задумывалось изначально.

Заодно добавлена диагностика — пункт настроек **«Проверить подключение к Trakt»** (виден только при включённом тумблере): по нажатию форсирует запрос к Trakt и показывает результат (найдены ли `client_id`/токен, успех с числом фильмов/сериалов, либо текст ошибки — `describeAjaxError()` достаёт HTTP-статус из jqXHR). `_traktLastFetch` хранит результат последней попытки, чтобы не гадать вслепую при повторных жалобах на статусы.

В watched-индексе теперь хранится `traktId` каждого сериала (для этого резолва), а бесполезная попытка собрать `episodes` из watched убрана. **Важно:** Trakt отслеживает серии бинарно (просмотрено/нет), без mid-episode percent — поэтому при включённых статусах Trakt в списке серий вместо процента Plex показывается метка «Просмотрено (Trakt)» либо пусто; resume-процент — это концепт Plex/Timeline, у Trakt его нет.

**Переименование кнопки + позиция воспроизведения (resume)** — два запроса пользователя:
1. Кнопка «Смотреть в **в** Plex» → «Смотреть **из** Plex» (Plex в Lampa — это источник воспроизведения, а не место, куда переходят). 2. **Resume не работал** — фильм/серия с частичным прогрессом (напр. 54%) начинались с начала, и во встроенном, и во внешнем плеере. Изучено, как это сделано в LampaTrakt: он **не вызывает `Player.play` напрямую** — полагается на нативное воспроизведение торрентов Lampa и лишь записывает позицию в хранилище `file_view_<profile>` по хэшу файла (`setFileViewProgress`), чтобы нативный плеер сам подхватил её. Наш случай иной (мы зовём `Player.play` со своим URL и объектом `timeline`), поэтому для внутреннего плеера resume должен работать через `data.timeline.time`. Вероятная причина сбоя — повторный запрос `Api.metadata` иногда возвращал элемент без `viewOffset`. Фикс: `playMeta(meta, cardData, playlistMetas, resume)` — новый параметр `resume {viewOffset, duration}` (в мс) с приоритетом над `meta.viewOffset`; для серий он берётся напрямую из элемента списка (`episodeStub`), по которому пользователь и видел процент — позиция гарантированно совпадает с показанной. Внутренний плеер перематывает по `timeline.time`, Android/webOS-внешние плееры получают позицию из `data.timeline.time` (Lampa проставляет `data.position` из неё). **Ограничение Apple TV:** для внешних плееров Apple TV (infuse/tvos и др.) сама Lampa НЕ передаёт стартовую позицию в URL-схеме (`infuse://…play?url=…`, `lampa://video?player=tvos&src=…` — параметра позиции там нет), в отличие от Android/webOS; resume в них зависит от собственной памяти плеера (infuse помнит позицию по URL). Это ограничение платформы/Lampa, не плагина.

**Фикс resume для внешних плееров Apple TV (tvOS Pro/infuse) — предыдущий вывод об «ограничении Apple TV» был НЕВЕРНЫМ.** Пользователь резонно заметил: при воспроизведении торрентов на tvOS Pro resume работает (стартует с середины). Значит позиция ДО плеера доходит, просто не через тот механизм, что я думал. Разобрано в `src/interaction/torrent.js` (нативное воспроизведение торрентов): воспроизводимый `element` содержит `element.playlist`, куда пушится в т.ч. **сам текущий элемент**, и **каждый элемент playlist несёт собственный `timeline`** (`{title, url, timeline, season, episode, ...}`). Внешний плеер (tvOS Pro/infuse) сопоставляет проигрываемый `src`-url с элементом playlist по url и берёт стартовую позицию из его `timeline` — позиция едет внутри `playlist` (сериализуется в URL-схему как `&playlist=`), а НЕ в отдельном параметре позиции. Наш баг: мы клали в `playData.playlist` только СЛЕДУЮЩИЕ серии, без текущего элемента и без `timeline` у элементов — плееру нечего было сопоставить. Фикс: `buildPlexPlaylistItem(meta, resume)` собирает элемент `{title, url, timeline, subtitles}` (timeline через `Lampa.Timeline.update`+`view` по позиции из resume/meta), а `playMeta` формирует `playlist`, начиная с **текущего** элемента (его `url === playData.url`), затем следующие серии — у каждой свой `timeline`. Теперь позиция доезжает до внешних плееров Apple TV так же, как у торрентов. Также это чинит корректность навигации по плейлисту (раньше текущего элемента в playlist не было вовсе).

**Отправка просмотра из Plex в Trakt** — по запросу пользователя. Решён архитектурный вопрос «где делать»: на стороне **Lampa-Plex**, т.к. только наш плагин знает в момент воспроизведения, что именно играет (tmdb id + сезон/серия); LampaTrakt про Plex-воспроизведение не знает (его скробблинг привязан к торрент-файлам через кэш хэшей), и учить его этому — лишняя связанность + правки чужого репозитория. Токен обновляет сам LampaTrakt (пишет свежий в `Storage`), мы читаем текущий. Реализация: тумблер `plex_trakt_scrobble` («Отправлять просмотр в Trakt», виден при установленном LampaTrakt). В `initTimelineSync` на `Player 'destroy'` для каждого реально просмотренного элемента (`road.time>0`) вызывается `reportPlaybackToTrakt`: при ≥90% — `POST /sync/history` (отметка «просмотрено»), ниже — `POST /scrobble/pause` (позиция для resume, best-effort). `traktPost` — POST-версия `traktRequest`. Идентификатор для Trakt (`{type:'movie',tmdb}` / `{type:'episode',tmdb:<show>,season,number}`) прокидывается через `playMeta(...,trakt)`: для фильмов из `injectPlexButton` (`e.data.id`), для серий из `showMeta`+`episodeStub` (`parentIndex`/`index`). **Очередь повтора** (`plex_trakt_scrobble_queue` в `Storage`): при сбое `POST /sync/history` payload сохраняется и до-отправляется на старте (`flushTraktQueue`, отложен 5с) — на прямой вопрос пользователя «при каком следующем разе, пересматривать?» ответ: НЕТ, повтор автоматический, пересмотр не нужен. После успешной отметки — `invalidateTraktCaches()` (сброс `_traktWatchedIndex`/`_traktEpisodeSetCache`), чтобы интерфейс плагина сразу показал новый статус. **Бинж-просмотр**: `_activePlexPlayback.items` хранит `{ratingKey,trakt}` по всем элементам плейлиста (текущий + следующие серии), поэтому при автопереходе отмечаются все просмотренные серии, а не только первая (актуально для внутреннего плеера; внешние обычно не отдают попутный прогресс назад).
